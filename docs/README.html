01

User & Access Management

Automated creation of a devops administrative user across all managed nodes, with sudo privileges and SSH key authentication deployed via the users role.

  • Create the devops group and user with a home directory
  • Add the user to the wheel group for sudo access
  • Deploy the SSH public key to authorized_keys
  • Verify account and SSH configuration across all nodes
02

Web Server Setup — Apache

Apache installed and configured with a dedicated virtual host, self-signed HTTPS, and firewall rules — deployed identically to Web1 and Web2.

  • Install & enable httpd, open HTTP/HTTPS via Firewalld
  • Deploy custom document root via Jinja2 virtual host template
  • Generate a self-signed SSL certificate and enable mod_ssl
  • Restore SELinux context on web content
03

Database Server — MariaDB

MariaDB installed and hardened: anonymous users and the test database removed, a dedicated application database and user provisioned with scoped privileges.

  • Install, enable, and start mariadb-server
  • Remove anonymous users and the default test database
  • Create application database rhcedb and user rhceuser
  • Grant scoped privileges and verify via SHOW GRANTS
04

Network File Sharing — NFS

A centralized NFS export mounted on both web servers, verified with a live write-then-read test between the file server and Web1.

  • Install nfs-utils, configure /etc/exports
  • Export /shared with rw,sync,no_root_squash
  • Mount the share on web servers and confirm via df -h
05

Security Hardening

Layered hardening across SELinux, Firewalld, and SSH — enforcing mode confirmed, minimal service exposure, and root login disabled.

  • SELinux verified in Enforcing mode on all nodes
  • Firewalld scoped to required services only (http, https, ssh)
  • PermitRootLogin no enforced in sshd_config
06

Load Balancing — HAProxy

HAProxy distributes HTTP traffic across Web1 and Web2 in round-robin, confirmed by repeated requests alternating between backends.

  • Configure frontend http_front bound to *:80
  • backend webservers with balance roundrobin
  • Verified alternating responses in the browser across refreshes
07

Monitoring & Logging

Prometheus scrapes Node Exporter on every managed node, Grafana visualizes the metrics, and rsyslog centralizes system logs on the Control Node.

  • Node Exporter deployed as a systemd service, port 9100 opened on all nodes
  • Prometheus configured with per-role scrape jobs (web, database, fileserver, haproxy)
  • Grafana connected to Prometheus and loaded with the Node Exporter dashboard
  • rsyslog forwards all managed-node logs to the Control Node on 514/udp
08

Cron Jobs & Maintenance

Website files and the MariaDB data directory are archived to /backup and the job is scheduled nightly via cron.

  • Archive /var/www/html → web_backup.tar.gz
  • Archive /var/lib/mysql → db_backup.tar.gz
  • Scheduled daily at 2:00 AM, output logged to /var/log/backup.log
# crontab -l
0 2 * * * cd /home/ec2-user/rhce-project && /usr/bin/ansible-playbook playbooks/backup.yml >> /var/log/backup.log 2>&1
§ 05

Getting Started

Clone the repository

$ git clone https://github.com/nandusivadas/RHCE-Capstone-Project.git
$ cd RHCE-Capstone-Project

Verify connectivity to all managed nodes

$ ansible all -i inventory/hosts -m ping

Run the main playbook

$ ansible-playbook -i inventory/hosts playbooks/site.yml

Or generate inventory dynamically from AWS EC2

$ ansible-inventory -i inventory/aws_ec2.yml --graph
